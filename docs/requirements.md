# 【2】要件定義書（全文・人間用）

> ※ 非エンジニア・意思決定者向け

---

## 前提（受け入れ不可なら中止すべき条件）

- Google広告のみを集客チャネルとする
- 商標・公式表現は全面禁止
- 「売れるか」より「死なないか」を優先する
- 判断は人間の裁量に委ねない

---

## 背景と課題

PPCアフィリエイトは以下の理由で再現不能・属人化・短命化してきた。

| 問題 | 説明 |
|------|------|
| 商標依存構造によるBAN | 商標KWに依存した広告運用 |
| 感覚的ジャンル選定 | データではなく勘で判断 |
| 成果ベースの後追い判断 | 事後的な評価に依存 |
| 「今回は大丈夫」という現場判断 | 属人的な例外判断 |

---

## 本システムの本当の目的

このシステムは
**アフィリエイトサイトを作るためのツールではない。**

> **「出してはいけない世界」を事前に確定させるための意思決定装置である。**

---

## 従来との決定的な違い

| 従来 | 本システム |
|------|------------|
| 売れそうか | 生き残れるか |
| 人が判断 | データが判断 |
| 改善前提 | 事故防止前提 |
| 結果主義 | 構造主義 |

---

## 作業フロー（人間の役割固定）

```
人間
  ↓ 開始指示
システム
  ↓ PPCリサーチ分析
  ↓ スコア算出
  ↓ 採否判定
人間
  ↓ 承認 or 却下
```

**途中判断は禁止。**

---

## 成果物の定義

1. **PPCリサーチ分析レポート**
2. **採否判定（Yes / No）**
3. **推奨比較サイト構造**
4. **Google広告用LPテンプレート（構造のみ）**

---

## 非機能要件

| 要件 | 説明 |
|------|------|
| 判断理由が説明可能 | すべての判定に根拠を提示 |
| 再実行で結果不変 | 同一入力→同一出力 |
| 個人差ゼロ | 誰が実行しても同じ結果 |
| 属人知識不要 | 専門知識なしで運用可能 |
