# 要件定義書（全文・人間用）

用途：理解・合意・運用判断
配置：docs/requirements.md

## 1. 前提（Assumptions）

本プロジェクトは、
Google広告を長期・安定運用するための
**商標レス意思決定支援サイト** を構築する。

一般的なアフィリエイトサイトとは目的が異なる。

## 2. 参考にしてよい／してはいけないもの

### 参考にしてよい

- 条件別・悩み別の意思決定支援メディア
- 診断・フローチャート型コンテンツ
- デメリット・注意点を明確に書く構成

### 参考にしてはいけない

- 商品名比較・ランキングサイト
- 公式画像・ロゴ多用LP
- 指名検索前提の広告事例

## 3. プロジェクトゴール

- 商標ゼロ構成のまま
- Google広告審査を通過
- 制限・警告なしで配信開始
- 案件差し替えが構造的に可能

※ 売上最大化・スケールは次フェーズ

## 4. 案件ジャンル選定要件

### Must

- 商標不要で説明可能
- 比較軸が複数存在
- 悩み・条件起点
- 向き不向き・注意点が書ける
- 法令・ポリシーリスクが低い

### Must Not

- 商品名検索が主流
- 効果断定が前提
- ブランド力依存
- 指名検索前提の高単価案件

## 5. A8案件採用要件

### Must（全Yes必須）

- Google広告利用可
- 商標使用不要
- 比較・デメリット記載可
- 診断・条件分岐に組み込める
- LP直リンク強制なし

### Must Not（1つで即NG）

- 商品名訴求前提
- 商標KW必須
- 公式画像必須
- 比較・デメリット禁止

## 6. 案件スコアリング

| 項目 | 点 |
|------|-----|
| 商標非依存 | +3 |
| 比較軸の多さ | +2 |
| デメリット記載可 | +2 |
| 広告自由度 | +2 |
| 法令リスク低 | +1 |

**合計8点以上のみ採用**

## 7. UX思想

- 主語は常にユーザー
- 条件・向き不向きを明示
- 判断はユーザーに委ねる
- メリット・デメリット両面必須

## 8. 作業フローと判断責任

| 工程 | 判断者 | NG基準 |
|------|--------|--------|
| ジャンル選定 | 発注者 | 商標依存 |
| 案件採用 | 発注者 | 8点未満 |
| 比較コピー | 発注者 | 商品名必須 |
| 画像 | ディレクター | 公式感 |
| 広告文 | 発注者 | LP乖離 |

## 9. Definition of Done

- 商標チェック工程が不要
- 全ページで 比較 / 診断 / 注意点 / CTA が成立
- Google広告審査通過
- 制限・警告なし

## 10. ディレクター向け注意

これは普通のアフィ案件ではない。
CVを急ぐ最適化は失敗である。
